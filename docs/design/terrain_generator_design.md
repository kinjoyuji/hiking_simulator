# TerrainGenerator モジュール 設計書

**バージョン**: 1.0.0
**対象ファイル**: `scripts/terrain/terrain_generator.gd`
**作成日**: 2026-02-24

---

## 1. 概要

国土地理院の標高タイル API から CSV を取得し、
Godot の `SurfaceTool` で歩行可能な 3D 地形メッシュをランタイム生成するクラス。

---

## 2. 依存する外部リソース

| リソース            | URL パターン                                                          | フォーマット             |
|--------------------|----------------------------------------------------------------------|--------------------------|
| 標高タイル (dem5a)  | `https://cyberjapandata.gsi.go.jp/xyz/dem5a/{z}/{x}/{y}.txt`        | CSV 256×256, 単位: m     |
| 標高タイル (dem)    | `https://cyberjapandata.gsi.go.jp/xyz/dem/{z}/{x}/{y}.txt`          | CSV 256×256（低解像度）  |

### フォールバック戦略

1. dem5a (ズーム≤15) を優先取得
2. 404 の場合は dem (ズーム≤14) にフォールバック
3. それでも失敗した場合はエラーログを出力し処理を中断

---

## 3. タイル座標変換（Webメルカトル）

```
n = 2^zoom
tile_x = int((lon + 180) / 360 * n)
lat_rad = radians(lat)
tile_y = int((1 - log(tan(lat_rad) + 1/cos(lat_rad)) / π) / 2 * n)
```

---

## 4. メッシュ生成パイプライン

```
[HTTP取得]
    ↓  response body (UTF-8 text)
[CSVパース]
    ↓  Array[Array[float]]  (256 × 256)
    ※ 欠損値 "e" → 0.0 で補完（TODO: 近傍補間に改善）
[頂点生成]
    ↓  SurfaceTool.add_vertex()
    ※ 水平スケール: 2.0 m/sample
    ※ 垂直スケール: 1.0 (そのままメートル)
[インデックス生成]
    ↓  2三角形 × (255 × 255) クアッド = 130,050 ポリゴン
[法線計算]
    ↓  SurfaceTool.generate_normals()
[コリジョン付与]
    →  MeshInstance3D.create_trimesh_collision()
```

---

## 5. メモリ使用量の見積もり

| 項目              | 計算                                   | 概算     |
|-----------------|----------------------------------------|---------|
| CSV raw テキスト  | 256 × 256 × 7 bytes/cell              | ~460 KB |
| パース済み配列    | 256 × 256 × 8 bytes (float64)         | ~524 KB |
| 頂点バッファ      | 256 × 256 × (pos+uv+normal) × 4bytes | ~2.4 MB |
| インデックスバッファ | 255×255×6 × 4bytes                | ~1.6 MB |
| **合計（1タイル）** |                                     | **~5 MB** |

4タイルロード時でも ~20MB となり許容範囲内。

---

## 6. 非同期処理の設計

```
generate_from_tile(z, x, y)
  │
  ├─ HTTPRequest.request(url)         # ノンブロッキング
  │
  └─ [コールバック] _on_request_completed()
       ├─ CSV パース
       ├─ メッシュ生成
       └─ add_child(mesh_instance)    # シーンツリーに追加
```

複数タイルを同時取得する場合は `_pending_tiles` 辞書で url → tile_key を管理し、
完了したものから順次メッシュを生成する。

## 7. 公開メソッド

| メソッド                                       | 説明                                |
|----------------------------------------------|-------------------------------------|
| `generate_from_tile(zoom, tile_x, tile_y)`   | 非同期でタイルを取得・メッシュ生成   |
| `latlon_to_tile(lat, lon, zoom) → Vector2i`  | 緯度経度 → タイル座標変換（純粋関数）|

---

## 8. エラー処理方針

| エラー条件              | 対処                                        |
|------------------------|---------------------------------------------|
| HTTP 404               | dem にフォールバック後、再試行              |
| HTTP 5xx / タイムアウト | `push_error()` でログ出力、処理中断        |
| CSVパース失敗           | 当該行をスキップ、0.0 で埋める             |
| メモリ不足              | Godot の組み込みエラーに委ねる            |

## 9. 既知の制約・将来拡張

| 項目              | 現在の制約                     | 将来の拡張               |
|-----------------|-------------------------------|--------------------------|
| タイル数         | 1タイルのみ                    | ストリーミング（4方向隣接） |
| 欠損値補完       | 0.0 固定                      | 近傍線形補間              |
| テクスチャ       | なし（白色メッシュ）           | 衛星写真テクスチャ合成     |
| LOD              | なし                          | 距離によるポリゴン削減     |
