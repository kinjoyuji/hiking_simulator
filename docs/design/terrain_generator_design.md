# TerrainGenerator モジュール 設計書

**対象ファイル**: `scripts/terrain/terrain_generator.gd`

---

## 1. 概要

国土地理院の標高タイル API から CSV を取得し、Godot の `SurfaceTool` で
歩行可能な 3D 地形メッシュをランタイム生成するクラス。
併せて低ポリの木・岩を `MultiMesh` で散布し、山の景観を構成する。
生成完了は `terrain_ready` シグナルで通知する。

---

## 2. 依存する外部リソース

| リソース            | URL パターン                                                  | フォーマット         |
|--------------------|--------------------------------------------------------------|----------------------|
| 標高タイル (dem5a)  | `https://cyberjapandata.gsi.go.jp/xyz/dem5a/{z}/{x}/{y}.txt` | CSV 256×256, 単位: m |

### フォールバック戦略

取得失敗（リクエスト発行エラー / 非200応答）時は、`FastNoiseLite` による
**仮想の山（シード固定）** を生成してゲームを成立させる。
オフライン環境・API 障害時でもプレイと自動テストが可能。
`terrain_ready` の `is_fallback` フラグで呼び出し側が判別できる。

---

## 3. タイル座標変換（Webメルカトル）

```
n = 2^zoom
tile_x = int((lon + 180) / 360 * n)
lat_rad = radians(lat)
tile_y = int((1 - log(tan(lat_rad) + 1/cos(lat_rad)) / π) / 2 * n)
```

`tools/fetch_elevation.py` の `lat_lon_to_tile` と同一式。テスト期待値も同式で算出する。

## 4. メッシュ生成パイプライン

```
[HTTP取得]
    ↓  response body (UTF-8 text)
[CSVパース]
    ↓  Array[Array[float]]  (256 × 256)
    ※ 欠損値 "e" → 0.0 で補完（TODO: 近傍補間に改善）
    ※ 空行はスキップ
[最小/最大標高の走査]
    ↓  min_h（正規化基準・登山口候補）, max_h（山頂）
[頂点生成]
    ↓  SurfaceTool.set_color() + add_vertex()
    ※ 水平スケール: 3.9 m/sample
       — zoom15タイル一辺 ≈ 1000m（緯度35.6°）/ 256サンプル。
         実寸に合わせないと傾斜角が誇張され、勾配倍率テーブルの前提が崩れる
    ※ 垂直: (標高 - min_h) をそのままメートルで使用（最低点が y=0）
    ※ 頂点カラー: 標高比で 緑(森) → 茶(岩) → 灰(頂) をグラデーション
[インデックス生成]
    ↓  2三角形 × (255 × 255) クアッド = 130,050 ポリゴン
    ※ 巻き順は法線が +y（上向き）になる順で固定。
      逆順だと描画・コリジョンとも裏返り、上からのレイ・落下物が素通りする
[法線計算]
    ↓  SurfaceTool.generate_normals()
[マテリアル]
    ↓  vertex_color_use_as_albedo + vertex_color_is_srgb
      （sRGB 指定しないと頂点カラーが明るく飛ぶ）
[コリジョン付与]
    ↓  MeshInstance3D.create_trimesh_collision()
[プロップ散布]
    →  _scatter_props()（木 1200・岩 250 を MultiMesh 配置）
[通知]
    →  terrain_ready.emit(info)
```

### terrain_ready の info 辞書

| キー              | 型      | 内容                                     |
|-------------------|---------|------------------------------------------|
| `spawn_position`  | Vector3 | 登山口（低標高かつ勾配≤15°の平坦地）     |
| `summit_position` | Vector3 | 山頂（タイル内側の最高標高点）           |
| `base_elevation`  | float   | y=0 に対応する実標高（HUD の標高表示用） |
| `max_elevation`   | float   | 実標高の最大値                           |
| `is_fallback`     | bool    | 代替地形を使ったか                       |

登山口・山頂ともタイル縁から 8 セル以上内側から選ぶ
（縁に置くとプレイヤーがメッシュ外へこぼれ落ちる）。
最低標高点そのものは渓谷の急斜面であることが多いため、
登山口は「低い」かつ「局所勾配が緩い」セルを条件とする。

---

## 5. プロップ散布（プロシージャル素材）

| プロップ | メッシュ                 | 配置条件                          | 数量 |
|----------|--------------------------|-----------------------------------|------|
| 木（幹） | CylinderMesh 低ポリ・茶  | 標高比 < 0.75（森林限界）かつ勾配 < 32° | 1200 |
| 木（樹冠）| 円錐 (CylinderMesh)・緑 | 幹と同じ変換 + 上方オフセット      | 同上 |
| 岩       | SphereMesh 低ポリ・灰    | 標高比 ≥ 0.5 または勾配 ≥ 25°     | 250  |

- 乱数シード固定（固定ルート MVP のため毎回同じ配置）
- コリジョンなし（物理コスト削減。すり抜け可はMVPでは許容）

---

## 6. 非同期処理の設計

```
generate_from_tile(z, x, y)
  │
  ├─ HTTPRequest.request(url)         # ノンブロッキング
  │    └─ 発行エラー → _generate_fallback_terrain()
  │
  └─ [コールバック] _on_request_completed()
       ├─ 非200 → _generate_fallback_terrain()
       └─ 200   → CSVパース → _build_terrain() → terrain_ready
```

MVP は 1 タイルのみ（同時複数リクエストなし）。

---

## 7. 公開API

| メンバー                                      | 説明                                  |
|----------------------------------------------|---------------------------------------|
| `signal terrain_ready(info: Dictionary)`     | 生成完了通知（§4 の info を渡す）     |
| `generate_from_tile(zoom, tile_x, tile_y)`   | 非同期でタイルを取得・メッシュ生成     |
| `latlon_to_tile(lat, lon, zoom) → Vector2i`  | 緯度経度 → タイル座標変換（純粋関数） |

## 8. 既知の制約・将来拡張

| 項目          | 現在の制約                       | 将来の拡張                 |
|---------------|----------------------------------|----------------------------|
| タイル数      | 1タイルのみ                      | ストリーミング（4方向隣接） |
| 欠損値補完    | 0.0 固定                         | 近傍線形補間               |
| テクスチャ    | 頂点カラーのみ                   | 衛星写真テクスチャ合成      |
| LOD           | なし                             | 距離によるポリゴン削減      |
| dem フォールバック | なし（失敗時は仮想地形）     | dem5a → dem の多段フォールバック |
| プロップ      | コリジョンなし・種類2種          | 当たり判定・種類追加・登山道 |
