# 設計書: 樹林帯の質感（森の癒し）

対応マイルストーン: M3（樹林帯の質感）
関連: [クオリティアップ・ロードマップ](../roadmap/quality_up_roadmap.md)

---

## 1. 体験目標

> 森林限界より下では、木々の間を縫う登山道を歩き、緑と木漏れ日に包まれる「癒し」を感じる。
> 登るにつれて樹高が下がり視界が開け、森林限界を超えた瞬間に稜線の遠望が広がる。

参考画像（シャクナゲの登山道）から抽出した要素:

1. **視界を包む植生** — 目線の高さに葉があり、道の両側から緑が迫る。空は樹冠の隙間から見える
2. **踏み跡のある登山道** — 石混じりの土の道が植生の間を縫う。道標がある
3. **植生の階層** — 高木（樹冠）・低木（目線）・下草（足元）の3層
4. **標高による遷移** — 広葉樹 → 針葉樹 → ハイマツ → 岩礫帯（参考画像1〜2枚目の稜線部）

## 2. 現状の構造と制約

- 木 = 円錐+円柱のプリミティブ 1200本、岩 250個を `MultiMesh` 一括散布（`_scatter_props`）
- 森林限界は**標高比** `TREELINE_RATIO = 0.75`（タイル内相対値。実標高と無関係）
- 登山道なし。地面は標高比の頂点カラー（緑→茶→灰）のみ
- プロップにコリジョンなし（木をすり抜ける）

## 3. M3-1: 植生帯の実標高化

### 植生帯の定義

タイル内標高比ではなく**実標高（`base_elevation + y`）**で帯を決める。
帯の境界はコース定義（現状は `main.gd` の定数群）に持たせ、緯度・山域で調整可能にする。

```gdscript
# scripts/terrain/vegetation_zones.gd（新規・Resource化も可）
# 本州中部の目安値。高尾山コース（〜600m）では全域が BROADLEAF になる点に注意
const ZONES := [
    { "name": "broadleaf",  "max_elev": 1500.0 },  # 広葉樹帯: 明るい緑・丸い樹冠
    { "name": "conifer",    "max_elev": 2500.0 },  # 針葉樹帯: 濃い緑・円錐樹形（シラビソ等）
    { "name": "krummholz",  "max_elev": 2700.0 },  # ハイマツ帯: 樹高1m前後・這う緑
    { "name": "alpine",     "max_elev": INF    },  # 高山帯: 岩礫・草付き。木なし
]
```

- 帯境界は ±80m 程度の**ノイズで揺らす**（直線的な「等高線刈り」を避ける。
  実際の森林限界も尾根・風衝で凸凹している）
- 傾斜 35°超・岩礫（M3-5 のスプラット岩領域）には帯に関わらず木を生やさない
- **この標高しきい値と揺らぎは M4 の音レイヤーがそのまま参照する**（森の音の減衰カーブ）。
  そのため M3-1 だけは M4 より先に完了させる

### テスト

- 境界値テスト: 各帯境界 ±1m で樹種が切り替わること・ノイズ揺らぎが ±80m を超えないこと
- 高尾山コース（全域広葉樹）とフォールバック地形（300〜560m だが仮想なので帯境界を
  下げた検証プリセットを用意）の両方で散布が成立すること

## 4. M3-2: 樹木アセット（プロシージャル + 2段LOD）

### 方針

外部モデリングツールは使わず、**エンジン内でプリミティブを組み合わせた低ポリ樹形**を作る。
「リアルな1本の木」より「森としての密度と色のばらつき」が癒し体験には効く。

| 帯 | 樹形 | 構成 |
|----|------|------|
| 広葉樹 | 丸い樹冠 | 幹（円柱）+ 樹冠（`SphereMesh` を2〜3個ずらして重ねる。高さ6〜12m） |
| 針葉樹 | 円錐2〜3段 | 幹 + 円錐を縦に2〜3段（現行の改良。高さ8〜15m） |
| ハイマツ | 低い塊 | つぶした `SphereMesh` の群れ（高さ0.5〜1.2m）。幹なし |
| 低木・下草 | クロスクアッド | 2枚交差の板ポリ + アルファ抜きテクスチャ（後述） |

- 色は帯の基準色に **HSVで ±8% のばらつき**を個体ごとに与える（`MultiMesh` の
  `use_colors = true` + `COLOR` をシェーダーで albedo に乗算）。単色の森は人工的に見える
- テクスチャ: 葉のアルファ抜きテクスチャは `Image` + `FastNoiseLite` で実行時生成するか、
  自作1枚（128×128）を `assets/textures/` に置く。外部素材を使う場合はCC0のみ

### 2段LOD

| LOD | 距離 | 表現 |
|-----|------|------|
| 近景 | 0〜150m | 上記メッシュ |
| 遠景 | 150〜600m | ビルボード板ポリ1枚（色は樹冠色。`BaseMaterial3D.BILLBOARD_FIXED_Y`） |
| それ以遠 | 600m〜 | 描画しない（地形の頂点カラー緑が「遠くの森」を担う） |

- 実装は **チャンク単位**（後述 M3-3）の `MultiMeshInstance3D` を近景用・遠景用の2つ持ち、
  `visibility_range_begin/end` + `visibility_range_fade_mode` でクロスフェードする
- 個体単位のLOD切替（MultiMesh内での距離判定）はしない（CPUコスト高。チャンク単位で十分）

## 5. M3-3: 密度アップと風揺れ

### 密度とチャンク分割

「森の中にいる」体感には目安として**樹木 20,000〜40,000本**（1km²の樹林帯想定）が必要。
一括 MultiMesh では描画制御できないため、チャンク分割を同時に入れる。

```
地形タイル(256×256セル) を 8×8 チャンク（32セル ≈ 125m 四方）に分割
各チャンク = { 高木近景MM, 高木遠景MM, 低木MM, 下草MM } の4 MultiMesh
チャンクごとに visibility_range を設定:
  高木: 0-600m / 低木: 0-200m / 下草: 0-80m
```

- 散布は現行同様シード固定（`PROP_SEED`）。チャンク座標をシードに混ぜて再現性を保つ
- 散布密度は帯ごとに定義（広葉樹帯 > 針葉樹帯 >> ハイマツ帯）。道（M3-4）の上と
  その両側1.5mは除外
- **パフォーマンス予算**（ロードマップ §4）: 描画対象インスタンス 1.2万以下/フレームを
  スモークテスト定点で計測

### 風揺れシェーダー

樹冠・下草のマテリアルを `ShaderMaterial` にし、頂点シェーダーで揺らす。

```glsl
// canopy_sway.gdshader（要点のみ）
uniform float sway_strength = 0.15;
uniform float sway_speed = 1.2;
void vertex() {
    float phase = INSTANCE_CUSTOM.x * 6.283;  // 個体ごとの位相（散布時にCUSTOMへ格納）
    float sway = sin(TIME * sway_speed + phase) * sway_strength;
    // 上ほど大きく揺れる（モデル空間yで重み付け）
    VERTEX.xz += vec2(sway, sway * 0.6) * clamp(VERTEX.y / 8.0, 0.0, 1.0);
}
```

- 風の強さ uniform は M4 の風音レイヤーと**同じパラメータを共有**する
  （GameManager が保持する `wind_intensity` を視覚と聴覚の両方に配る）— 音と絵の一致が没入感を作る

### コリジョン

- 高木の幹のみ、**プレイヤー近傍（半径20m）に動的生成**する軽量方式を採る:
  毎フレームではなく2秒ごとに周辺チャンクの幹位置から `CylinderShape3D` を再配置（プール10本程度）
- 全木にStaticBodyを付けるのは物理負荷とメモリの無駄（CLAUDE.md のフェーズ2予定
  「プロップの当たり判定」はこの限定方式で満たす）

## 6. M3-4: 登山道の生成

### パス計算

登山口（spawn）→ 山頂（summit）を `AStarGrid2D`（Godot組み込み）で結ぶ。

```gdscript
# scripts/terrain/trail_planner.gd（新規）
# コスト = 距離 × 傾斜ペナルティ。実際の登山道が九十九折りになる理屈をそのまま使う
#   slope_penalty = 1.0 + pow(slope_deg / 12.0, 2.0)   # 12°を快適勾配とする
# → 急斜面を直登せず、等高線に沿って巻くパスが自然に出る
```

- 出力: セル列 `Array[Vector2i]` → Catmull-Rom で平滑化してワールド座標のポリラインへ
- `terrain_ready` の info に `trail_points: PackedVector3Array` を追加（HUD のミニマップ・
  M4 の「道から離れた度合い」判定でも使う）

### 道の描画

- **地形メッシュの頂点カラーを道の色（明るい土色）に上書き**する方式を採る
  （デカール不要・追加ドローコールなし。3.9m グリッドで道幅2〜4mをカバーできる）
- パスから距離1.5セル以内の頂点を土色に、1.5〜2.5セルをブレンドで馴染ませる
- M3-5 のスプラットシェーダー導入後は「道マスク」を頂点カラーの α チャンネルに移し、
  シェーダー側で土テクスチャを出す
- 道沿いの演出プロップ: 道標（分岐・山頂前）、ケルン（森林限界以上）を数個置く。
  参考画像2枚目の「標石+道」の記号性は少数のプロップで十分出る

### テスト

- 機能テスト: パスが spawn と summit を結ぶ・全長が直線距離の1.2〜4倍に収まる
- 境界値テスト: 最大傾斜セルだけで囲まれた場合でもパスが見つかる（コスト無限大にしない）

## 7. M3-5: 地面スプラットシェーダー

頂点カラー1色 → **傾斜・標高・道マスクで4種をブレンド**するシェーダーへ。

| レイヤー | 条件 | 見た目 |
|---------|------|--------|
| 草・下生え | 樹林帯 & 傾斜 < 30° | 緑（帯でトーン変化） |
| 土 | 道マスク | 明るい茶。石の細かいノイズ |
| 岩盤 | 傾斜 ≥ 35°（ブレンド幅5°） | 灰。崖・ガレの表現 |
| 岩礫（ガレ） | 高山帯 & 傾斜 < 35° | 明るい灰茶（参考画像1枚目のザレ場） |

- テクスチャは 256×256 の手書き/ノイズ生成各1枚 + ノーマルなし（頂点法線で十分）から始める
- 遠距離ではテクスチャをフェードして頂点カラー相当の単色に戻す（タイリングの繰り返し感対策）
- **移行期の保険**: `use_splat_shader` フラグで旧頂点カラーへ即戻せるようにする（ロードマップ §5）

| 順 | タスク | 検証 |

| 3 | M3-2 樹形バリエーション + LOD | 定点スクリーンショット・描画インスタンス数計測 |

| 5 | M3-4 登山道 | パス生成のGUTテスト + 歩行して道を辿れるか目視 |

## 9. 決めごと

- 樹木の外部3Dモデル導入は**このフェーズではしない**。プロシージャル樹形で密度を優先し、
  1本のリアリティはM5以降に評価する
- 下草・低木のコリジョンは付けない（歩行の邪魔になり、癒しに逆効果）
- 木漏れ日（ボリュメトリックライト）は M5-2 へ後送り。ここでは樹冠の影
  （`DirectionalLight3D` の shadow）が地面に落ちるだけで雰囲気は大きく変わる
