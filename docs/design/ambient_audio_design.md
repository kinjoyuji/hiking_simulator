# 設計書: 環境音の重層化

対応マイルストーン: M4（音の重層化）
関連: [クオリティアップ・ロードマップ](../roadmap/quality_up_roadmap.md)

---

## 1. 体験目標

> 目を閉じても「いま樹林帯にいるか、稜線にいるか」が分かる。
> 標高・植生・天候・自分の行動（歩く/立ち止まる）に応じて音が重なり合い、その場の空気を作る。

体験の設計原則:

1. **場所が音を決める** — 標高（風の強さ）・植生帯（鳥・葉擦れ）・地表（足音）・水辺（沢音）
2. **立ち止まると音が主役になる** — 休憩（体力回復）中は足音が消え、環境音だけが残る。
   「立ち止まって休む」という既存メカニクスに聴覚の報酬を与える
3. **音は連続的に遷移する** — 帯の境界でスパッと切り替わらず、標高に応じてクロスフェード

## 2. 現状の構造と制約

- `WindPlayer`（`AudioStreamPlayer`・グローバル）で合成風ループ1本を `-12dB` 固定再生
- バスは Master のみ。3D音源・ワンショット・足音なし
- 音素材は `tools/generate_ambient.py` による自前合成WAV 1個（外部依存ゼロ）

## 3. バス構成（M4-1）

```
Master
 ├── Ambient        … 環境音ループ層（風・森・沢）
 │    └── (AudioEffectReverb: 谷・樹林で室感を微調整。稜線ではほぼドライ)
 ├── SFX            … 足音・飲水・UI
 └── Weather        … 雨・雷（既存の雨イベントを音でも表現）
```

- `default_bus_layout.tres` を作成し `project.godot` に登録する
- リバーブは1個を薄く（wet 0.05〜0.15）。パラメータは場所係数（後述 §5）で可変
- 音量ミキシングの原則: **ベース環境音は -18〜-12dB に置き、ワンショットで前景を作る**。
  常時鳴る音を大きくすると疲れる（癒しの逆）

## 4. 音レイヤー定義（M4-2〜M4-4）

| レイヤー | 種別 | 再生方法 | 音量・音色を決める入力 |
|---------|------|---------|----------------------|
| 風（基本） | ループ | `AudioStreamPlayer`（2D） | 標高比↑で音量↑・ローパス開放（稜線ほど「ゴォー」が鋭く） |
| 風（樹冠の葉擦れ） | ループ | 同上 | 植生帯の樹木密度 × 風強度。樹林帯では風が「幹の間を抜ける音+葉擦れ」になる |
| 森の基調（鳥・虫の遠景） | ループ | 同上 | 樹木密度↑で音量↑。時刻で内容が変わる（朝: 鳥多め / 昼: セミ・静か / 夕: ヒグラシ） |
| 鳥の一声 | ワンショット | `AudioStreamPlayer3D` をプレイヤー周囲にランダム配置 | 樹林帯のみ。15〜60秒間隔・ランダム方位 20〜60m |
| 沢音 | ループ | `AudioStreamPlayer3D`（水場に固定） | 地形の谷線（後述）に配置。距離減衰で近づくと聞こえる |
| 足音 | ワンショット | `AudioStreamRandomizer` + `AudioStreamPlayer` | 地表種別（土/岩/ガレ）× 歩調（移動速度） |
| 雨 | ループ | Weather バス | 既存 `_on_weather_changed` でフェードイン。樹林帯では「葉を叩く音」に変化 |
| 突風 | ワンショット | 2D | 風強度が高い時にランダム。M3の樹冠揺れシェーダーの `wind_intensity` と同期 |

### 場所→音のパラメータフロー

```
AmbienceController (新規: scripts/audio/ambience_controller.gd)
    入力（毎秒程度のポーリングで十分。毎フレーム不要）:
      - elevation_ratio  … プレイヤー実標高と森林限界の関係（M3-1の植生帯APIを参照）
      - canopy_density   … 現在チャンクの樹木密度（0=裸地〜1=密林）
      - wind_intensity   … GameManagerが天候・標高から算出（視覚の樹冠揺れと共有）
      - time_of_day      … HUDのゲーム内時刻
      - is_moving / surface_type … Player から
    出力:
      - 各ループレイヤーの目標音量・ローパスカットオフ → Tween で1〜3秒かけて追従
      - ワンショットのスケジューリング
```

- **クロスフェードは目標値追従型**にする（イベント駆動で切り替えると境界の行き来で
  パタパタする）。`volume_db` は線形値でlerpしてからdB変換（dB直lerpは中間が凹む）
- 耐久テスト観点: 28800フレーム歩行でTweenの多重起動・音量のNaN/発散がないこと（DU系に追加）

### 足音（M4-3）

- 歩調: `WALK_SPEED 3.0m/s`・歩幅0.75m → 約0.5秒/歩。`is_moving` の間タイマーで発火
- 地表種別は M3-5 のスプラット判定を流用（道=土、傾斜35°超=岩、高山帯=ガレ）。
  M3-5 未完了の間は「植生帯」だけで土/ガレの2種から始める
- `AudioStreamRandomizer` に各種3〜4バリエーション + ピッチ揺らぎ ±5% を設定
  （同じ音の連打は最悪の没入破壊なので、バリエーションは最初から用意する）

### 沢音の配置

- 地形解析で**谷線**を抽出: 各セルから最急降下方向へ水を流すシミュレーション
  （flow accumulation）で流量の多いセル列を谷線とみなす
- 流量しきい値を超える谷線に 50m 間隔で `AudioStreamPlayer3D`（max_distance 60m）を配置
- 視覚の川メッシュは作らない（M5以降）。「音だけの沢」でも森の奥行きは大きく増す
- フォールバック地形でも同じ解析がそのまま動く

## 5. 音素材制作パイプライン（M4-5）

### 方針: 自前合成ファースト

`tools/generate_ambient.py` を `tools/audio/` パッケージに発展させ、全素材を
シード固定の合成で再現可能に保つ（ライセンス管理ゼロ・ビルド再現性維持）。

| 素材 | 合成方法（要点） |
|------|----------------|
| 風・稜線 | 現行方式 + カットオフを上げ、突風LFOを強く |
| 風・樹林（葉擦れ） | ホワイトノイズを高めの帯域（2〜4kHz）で薄く、粒状の振幅変調（葉の粒感） |
| 沢 | ノイズ2層（低域「ゴボゴボ」+ 高域「サラサラ」）に速い振幅揺らぎ |
| 足音・土 | 短いノイズバースト（30〜60ms）+ ローパス + 減衰エンベロープ |
| 足音・岩/ガレ | 上記 + 高域成分 + 「小石が転がる」後続の微小バースト2〜3個 |
| 雨 | 密度の高いランダムインパルス列 + ローパス。樹林版は帯域を上げ「葉を叩く」感じに |
| 鳥の一声 | サイン波のピッチエンベロープ（チャープ）2〜3音 + 倍音。数種類のパターンをシードで量産 |

- 各生成器は `--seed` と `--out` を持ち、`make audio`（または `tools/audio/generate_all.py`）で
  `assets/audio/` を一括再生成できるようにする
- 生成物はリポジトリにコミットする（Godotプロジェクトを開くだけで音が鳴る状態を保つ。
  再生成スクリプトが「ソース」、WAVは「ビルド成果物だが利便性のためコミット」という扱い）

### 外部素材（CC0）の導入基準

合成で品質が出ないと判断した素材（特に鳥のさえずり・セミ）のみ外部を検討する。

- **CC0（パブリックドメイン）限定**。CC-BYは帰属表示管理が漏れやすいので使わない
- 入手先の例: freesound.org（ライセンスフィルタでCC0指定）
- 導入時は `assets/audio/CREDITS.md` に「ファイル名 / 出典URL / 取得日 / ライセンス」を記録
- 加工（ループ化・帯域調整）は `tools/audio/` にスクリプトを残す

### Godot側のインポート設定

- ループ素材は `.wav` のインポート設定でループ有効化するか、現行どおり実行時に
  `loop_mode` を設定（`main.gd` `_start_wind_ambience` の方式）。**素材が増えるため
  インポート設定（`.import` ファイル）でのループ指定に統一**し、実行時duplicateを廃止する
- 22050Hz モノラルで統一（環境音に高サンプルレート不要。3D音源はモノラル必須）

## 6. 実装順序と検証

| 順 | タスク | 検証 |
|----|--------|------|
| 1 | M4-1 バスレイアウト | バスが存在し各プレイヤーが割当済み（スモークテストで assert） |
| 2 | M4-5 合成ツール拡張（風2種・足音） | pytest: 生成WAVの長さ・ピーク・ループ境界の連続性 |
| 3 | M4-2 AmbienceController + 風・森レイヤー | GUT: 標高入力→目標音量のテーブルテスト（境界値: 森林限界±） |
| 4 | M4-3 足音 | 歩行/停止で発火が切り替わるGUTテスト + 目視（耳視）確認 |
| 5 | M4-4 鳥ワンショット・沢音 | 谷線抽出のユニットテスト（フォールバック地形で決定的） |
| 6 | 雨の音接続 | 既存天候イベントテストに音アサート追加 |

## 7. 決めごと

- BGM（音楽）は**入れない**。環境音だけで場を作るのがこのゲームの方向性
  （リザルト画面のジングル程度は将来検討可）
- `AudioStreamInteractive`（Godot 4.3+のトランジション機能）は使わず、
  自前の音量lerpで済ませる（レイヤー数が少なく、制御ロジックをテスト可能に保ちたい）
- 音の定位: 環境ループは2D（無定位）、個別イベントのみ3D。全部3Dにすると
  ヘッドホンで酔う・スピーカーで破綻する
- 立ち止まり休憩時は Ambient バスを +2〜3dB 持ち上げる演出を入れる（§1 原則2）。
  ただし体力回復レートには影響させない（ゲームバランスと演出の分離）
