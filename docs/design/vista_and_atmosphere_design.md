# 設計書: 遠景の山並みと空気感（稜線体験）

対応マイルストーン: M1（空気と光）・M2（遠景の山並み）
関連: [クオリティアップ・ロードマップ](../roadmap/quality_up_roadmap.md)

---

## 1. 体験目標

> 標高が上がるほど視界が開け、幾重にも重なる山並みが「遠くほど青く霞んで」どこまでも見える。

参考画像（天狗岳稜線・冬山山頂）から抽出した視覚要素:

1. **多重の稜線シルエット** — 近い尾根は緑/茶、遠い山脈ほど青→白へ段階的に霞む（空気遠近法）
2. **地平線まで続く地形** — プレイエリア外にも実在の山々が描かれ、「切れ目」が見えない
3. **空と霞の連続性** — 地平線付近の空は白っぽく、遠景の山はそこに溶け込む
4. **標高による視界の変化** — 樹林帯では遠景がほぼ見えず、森林限界を超えると一気に開ける（M3の植生と協調）

## 2. 現状の構造と制約

- `terrain_generator.gd` は **1タイル（z15, 256×256サンプル, 約1km四方）** のみ生成
- 標高は**タイル内最小値で y=0 に正規化**（`base_elevation`）→ 複数タイルを並べると基準がずれて崖ができる
- `Camera3D.far = 2000` — 1km四方には十分だが遠景には不足
- `Environment` はフォグなし・`ProceduralSkyMaterial`（散乱表現なし）

## 3. 全体構成

3層の同心構造でスケールと解像度を分担する。

```
        ┌─────────────────────────────────────┐
        │  遠景リング (z12前後, 〜半径30-60km)      │  低解像度メッシュ・コリジョンなし
        │   ┌───────────────────────────┐     │  空気遠近で空に溶ける
        │   │  近傍タイル 3×3 (z15, 3km四方) │     │  歩行可能・コリジョンあり
        │   │   ┌─────────────┐         │     │
        │   │   │ 中央タイル     │         │     │  ← 現在の生成範囲
        │   │   │ (プレイエリア)  │         │     │
        │   │   └─────────────┘         │     │
        │   └───────────────────────────┘     │
        └─────────────────────────────────────┘
```

| 層 | データ | 解像度 | メッシュ | コリジョン |
|----|--------|--------|---------|-----------|
| 中央+近傍 3×3 | `dem5a` z15 | 約3.9m/サンプル | フル（現行方式） | あり |
| 遠景リング | 低ズーム（`dem10b` z14 を間引き or より低いズームの `dem` 系） | 40〜160m/サンプル | 間引きグリッド | なし |

> **データセットの事前確認**: GSIの標高タイルはデータセットごとに提供ズームが異なる。
> 実装前に `python3 tools/fetch_elevation.py --lat 35.6252 --lon 139.2437 --zoom 12`
> のようにズームとデータセットを変えて疎通確認し、この表を実測で更新すること。
> （`fetch_elevation.py` に `--dataset` オプションを追加するのが最初のタスク）

## 4. M1: 空気と光（フォグ・空・ライティング）

地形を増やす前に、まず「空気」を作る。1タイルのままでも見た目が大きく変わり、
遠景リング導入後はそのまま多重稜線の霞になる。

### 4-1. Environment 設定（main.tscn）

```gdscript
# WorldEnvironment の Environment に設定する値（tscn直接編集でも可）
environment.background_mode = Environment.BG_SKY
environment.sky.sky_material = PhysicalSkyMaterial.new()  # レイリー/ミー散乱で地平線の霞が出る

environment.tonemap_mode = Environment.TONE_MAPPER_AGX    # 白飛びに強い。FILMICでも可
environment.tonemap_exposure = 1.0

environment.fog_enabled = true
environment.fog_mode = Environment.FOG_MODE_EXPONENTIAL
environment.fog_density = 0.0001          # ベースは極薄。天候で可変（雨: 0.001 目安）
environment.fog_aerial_perspective = 0.7  # 遠景の色を空の色へ寄せる = 空気遠近の核
environment.fog_sky_affect = 0.3          # 地平線付近の空も少し霞ませ、山と空を馴染ませる
environment.fog_light_color = Color(0.75, 0.82, 0.92)  # 昼の霞は青白
```

要点:

- **空気遠近の主役は `fog_aerial_perspective`**。単純な色フォグ（`fog_light_color` 一色に沈む）ではなく、
  距離に応じて空の色に溶けるため「青く霞む山並み」が出る
- `fog_density` は天候・時刻の演出パラメータとして GameManager から可変にする
  （既存の雨イベント `_on_weather_changed` で濃度を上げる → M5-3 の布石）
- `PhysicalSkyMaterial` の `mie_coefficient` / `turbidity` 相当パラメータで
  「澄んだ冬空」（参考画像4枚目）と「夏の霞んだ空」を作り分けられる。コース設定に持たせる

### 4-2. 時刻連動ライティング（M1-4）

HUD の時計はゲーム内時刻を持っている（`hud.set_clock_running`）。これを光源に接続する。

```gdscript
# GameManager が毎分（ゲーム内）呼ぶ
func _update_sun(game_hour: float) -> void:
    # 6時=日の出(-0°) 12時=南中(+60°) 18時=日没 の簡易モデル
    var t := clampf((game_hour - 6.0) / 12.0, 0.0, 1.0)
    var altitude := sin(t * PI) * 60.0
    sun.rotation_degrees.x = -altitude
    sun.rotation_degrees.y = lerpf(-100.0, +100.0, t)  # 東→西
    sun.light_color = Color(1.0, 0.75, 0.55).lerp(Color(1, 0.98, 0.95), sin(t * PI))
    sun.light_energy = lerpf(0.4, 1.3, sin(t * PI))
```

- 実装位置: `main.gd` に `_sun_pivot` を持たせ、HUDの時刻更新シグナル（要追加）を購読
- 朝スタート（06:00）の斜光は谷に長い影を落とし、地形の立体感を大きく上げる

### 4-3. カメラ・描画距離

- `Camera3D.far`: 2000 → **80000**（遠景リング半径+マージン。fog_aerial_perspective があれば
  ファークリップの「切れ」はほぼ見えないが、リングの外周は必ず霞の中に収める）
- `near` は 0.05 のまま（far拡大で深度精度が落ちるため、Forward+ の逆Z深度で問題ないか
  遠景導入時に Z-fighting を目視確認。問題があれば near を 0.1 へ）

## 5. M2: マルチタイル地形

### 5-1. 標高基準のグローバル化（M2-1・最初にやる）

現状の「タイル内最小標高 = y=0」はマルチタイルで破綻する。

**変更方針**:

- 正規化基準を「タイルごとの min」から「**コース基準標高 `course_datum`**（中央タイルの min を最初に採用し固定）」へ
- `_build_terrain` の署名を `_build_terrain(elevation_data, is_fallback, datum: float)` に拡張
- `terrain_ready` の `base_elevation` は `course_datum` を返す（HUD の実標高表示は互換のまま）
- 既存テスト（tests/unit/test_terrain_generator.gd）の正規化前提を更新

### 5-2. 近傍タイル 3×3（M2-2）

- `generate_from_tile(zoom, x, y)` を `generate_area(zoom, center_x, center_y, radius=1)` に拡張
- 各タイルを個別の `MeshInstance3D` として生成し、ワールド原点は中央タイルの北西角に固定。
  タイル (dx, dy) のオフセットは `Vector3(dx * 256 * MESH_SCALE, 0, dy * 256 * MESH_SCALE)`

**継ぎ目処理（重要）**: GSIタイルは256×256セルで、隣接タイルとの間に共有行/列がない。
そのまま並べると1セル分（約3.9m）の隙間ができる。

- 対策: タイル取得後に**東隣タイルの第0列・南隣タイルの第0行を自タイルの257列/行目として結合**し、
  各タイルを 257×257 頂点でメッシュ化する（グリッドが連続し、隙間も法線の割れも出ない）
- 欠損値 `"e"` の補間（現状 0.0 埋め）はマルチタイル化で目立つようになるため、
  このタイミングで**近傍平均補間**へ改善する（CLAUDE.md 記載の既知課題）

**座標系の注意**: `MESH_SCALE = 3.9` は緯度35.6°前提。汎用化する場合は
`meters_per_sample = 156543.03392 * cos(lat_rad) / 2^zoom / (256/タイル解像度)` で計算する。
（コース選択機能が入るまでは定数のままでよいが、コメントに式を残す）

### 5-3. 遠景リング（M2-3）

- 低ズームタイル1枚は広域をカバーする（z12なら1タイル≈8km四方@関東）。中央を囲む
  5×5 程度の低ズームタイルで半径 20〜40km をカバーできる
- メッシュは**4〜8サンプルに1頂点へ間引き**（遠景は霞むためディテール不要）
- **プレイエリアと重なる中央部は描画しない**（近景メッシュと Z-fighting するため、
  中央タイル領域に該当するクアッドをインデックス生成時にスキップ）
- マテリアルは近景と同じ頂点カラー方式でよい（霞が支配的になるため質感差は目立たない）
- `GeometryInstance3D.visibility_range_begin` は使わない（遠景は常時表示。
  近景の木などのLODにのみ使用）
- 読み込みは**非同期・後追い**: `terrain_ready`（中央タイル）でゲームを開始し、
  近傍→遠景の順にバックグラウンドで追加する。揃うまではフォグが外周を隠す

### 5-4. タイル取得基盤（M2-5）

現状の単発 `HTTPRequest` を小さなフェッチャークラスに分離する。

```
scripts/terrain/tile_fetcher.gd
  - request_tile(dataset, z, x, y) -> Signal(tile_loaded(key, data))
  - 同時実行 4 本まで・キュー管理
  - リトライ: 1回（指数バックオフ不要。失敗タイルはフォールバック生成 or 遠景ならスキップ）
  - ディスクキャッシュ: user://tile_cache/{dataset}/{z}/{x}/{y}.txt
    （2回目以降の起動は通信ゼロ。GSIサーバーへの負荷配慮でもある）
```

- キャッシュ破損（パース失敗）時はキャッシュ削除→再取得1回→ダメならフォールバック
- `terrain_generator.gd` はフェッチャーからのデータを受けてメッシュ化に専念する形に整理

### 5-5. フォールバック（オフライン）との整合

- フォールバック地形は現状どおり1枚生成とし、**遠景リングの代わりに fog_density を上げて**
  「霞で視界が閉じた日」として成立させる（機能欠落ではなく天候として説明がつく）
- スモークテスト（オフライン経路）はこのパスを通る。`terrain_ready` の契約は変えない

## 6. 実装順序と検証

| 順 | タスク | 検証 |
|----|--------|------|
| 1 | M1-1〜3 フォグ・空・トーンマップ | スクリーンショット比較（山頂から谷を見下ろす定点） |
| 2 | M1-4 時刻連動ライト | 06:00/12:00 の影の向き目視 + 単体テスト（角度計算） |

## 7. 決めごと（実装時に迷わないため）

- 遠景リングの半径は**まず z12×5×5 で作って見た目で判断**する。「地平線の切れ」が
  fog_sky_affect で隠れるなら広げない（データ取得量とのトレードオフ）
- 雪山表現（参考画像4枚目）はこのフェーズでは**やらない**。`_elevation_color` の
  peak 色を白に寄せるだけの季節プリセットは M5 以降の検討事項
- 遠景タイルのプロップ散布（木・岩）は**しない**（距離的に視認不能）
