# GameManager モジュール 設計書

**バージョン**: 1.0.0
**対象ファイル**: `scripts/main.gd`
**作成日**: 2026-02-24

---

## 1. 概要

ゲーム全体のフェーズ管理・ノード間協調・教育的フィードバック出力を担うルートコントローラ。
グループ名 `"game_manager"` で登録し、任意のノードから `call_group()` で参照可能。

---

## 2. フェーズ状態遷移

```
  ┌─────────────────────────────────────────────────────┐
  │                                                     │
  ▼                                                     │
[PLANNING] ──(start_hiking())──► [HIKING]              │
                                    │                   │
                     ┌──────────────┼───────────────┐   │
                     │              │               │   │
              (player_downed) (goal_reached)  (fatal event)
                     │              │               │   │
                     ▼              ▼               ▼   │
                 [RESULT]       [RESULT]       [EMERGENCY]─┘
```

### 各フェーズの仕様

| フェーズ    | 説明                                      | 入力受付 | 物理演算 |
|-----------|------------------------------------------|---------|---------|
| PLANNING  | ルート確認・装備選択UI表示               | UIのみ  | 停止    |
| HIKING    | プレイヤー操作・パラメータ消耗            | キーボード+マウス | 有効 |
| EMERGENCY | トラブルイベント処理（MVP未実装）         | UIのみ  | 停止    |
| RESULT    | 結果表示・教育フィードバック・リトライ    | UIのみ  | 停止    |

---

## 3. 責務分担（依存関係図）

```
GameManager
  ├── Player (CharacterBody3D)
  │     └── PlayerStats (Node)  ← シグナルで疎結合
  ├── TerrainGenerator (Node3D)
  ├── HUD (CanvasLayer)          ← PlayerStats をバインド
  ├── PlanningPanel (Control)
  └── ResultPanel (Control)
```

GameManager は各ノードへの参照を `@onready` で取得し、
ノード間の直接参照はすべて GameManager を経由させる（仲介パターン）。

---

## 4. 教育的フィードバック設計

ゲームオーバー時に「なぜ失敗したか」を具体的に提示する。

| 原因キーワード | フィードバックメッセージ              |
|-------------|--------------------------------------|
| "脱水"      | 水分補給タイミングの説明             |
| "体力"      | 引き返し判断・体力配分の重要性を説明 |
| その他       | 余裕ある計画の必要性を説明           |

---

## 5. 公開メソッド

| メソッド                          | 呼び出し元          | 説明                           |
|----------------------------------|--------------------|---------------------------------|
| `start_hiking()`                 | PlanningPanel      | 行動フェーズ開始               |
| `on_player_downed(reason)`       | Player (call_group) | ゲームオーバー処理             |
| `on_goal_reached()`              | GoalArea (将来)     | ゴール到達処理                 |

## 6. MVP との対応

| 機能              | MVP 実装状況    | フェーズ2以降               |
|-----------------|---------------|------------------------------|
| 計画フェーズ      | UI 表示のみ    | 装備選択・ルート検討UI        |
| コース選択        | 高尾山固定     | 複数コース選択画面           |
| 緊急フェーズ      | 未実装         | 道迷い・落石などのイベント   |
| リトライ          | シーン再読込   | 反省点を踏まえた条件引き継ぎ |
