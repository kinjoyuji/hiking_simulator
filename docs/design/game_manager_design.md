# GameManager モジュール 設計書

**対象ファイル**: `scripts/main.gd`

---

## 1. 概要

ゲーム全体のフェーズ管理・ノード間協調・ゴール設置・天候イベント・
教育的フィードバック出力を担うルートコントローラ。
グループ名 `"game_manager"` で登録し、任意のノードから `call_group()` で参照可能。

---

## 2. フェーズ状態遷移

```
  ┌─────────────────────────────────────────────────────┐
  │                                                     │
  ▼                                                     │
[PLANNING] ──(start_hiking())──► [HIKING]              │
                                    │                   │
                     ┌──────────────┼───────────────┐   │
                     │              │               │   │
              (player_downed) (goal_reached)  (fatal event)
                     │              │               │   │
                     ▼              ▼               ▼   │
                 [RESULT]       [RESULT]       [EMERGENCY]─┘
```

`start_hiking()` は即座に HIKING へ遷移せず、TerrainGenerator の
`terrain_ready` シグナルを待つ（HUD に「地形データを取得中…」を表示）。
地形完成後にプレイヤーを登山口へ配置し、山頂にゴールを設置してから HIKING に入る。

```
start_hiking()
  ├─ 計画パネル非表示・ローディング表示
  └─ terrain_generator.generate_from_tile(高尾山タイル)
        ↓ terrain_ready(info)
_on_terrain_ready(info)
  ├─ player を info.spawn_position へ配置
  ├─ 山頂 (info.summit_position) にゴールエリア + 赤い標柱を生成
  ├─ HUD に基準標高・目標を渡す（標高・山頂までの距離表示）
  ├─ 風の環境音を再生（ループ）
  ├─ 天候急変イベントを予約（60〜150秒後に雨 → 消耗×1.3）
  └─ HIKING へ遷移・マウスキャプチャ
```

### 各フェーズの仕様

| フェーズ    | 説明                                      | 入力受付 | 物理演算 |
|-----------|------------------------------------------|---------|---------|
| PLANNING  | ルート確認・装備選択UI表示               | UIのみ  | 停止    |
| HIKING    | プレイヤー操作・パラメータ消耗            | キーボード+マウス | 有効 |
| EMERGENCY | トラブルイベント処理（MVP未実装）         | UIのみ  | 停止    |
| RESULT    | 結果表示・教育フィードバック・リトライ    | UIのみ  | 停止    |

---

## 3. 責務分担（依存関係図）

```
GameManager
  ├── Player (CharacterBody3D)
  │     └── PlayerStats (Node)  ← シグナルで疎結合
  ├── TerrainGenerator (Node3D)
  ├── HUD (CanvasLayer)          ← PlayerStats をバインド
  ├── PlanningPanel (Control)
  └── ResultPanel (Control)
```

GameManager は各ノードへの参照を `@onready` で取得し、
ノード間の直接参照はすべて GameManager を経由させる（仲介パターン）。

---

## 4. 教育的フィードバック設計

ゲームオーバー時に「なぜ失敗したか」を具体的に提示する。

| 原因キーワード | フィードバックメッセージ              |
|-------------|--------------------------------------|
| "脱水"      | 水分補給タイミングの説明             |
| "体力"      | 引き返し判断・体力配分の重要性を説明 |
| その他       | 余裕ある計画の必要性を説明           |

---

## 5. 公開メソッド

| メソッド                          | 呼び出し元          | 説明                           |
|----------------------------------|--------------------|---------------------------------|
| `start_hiking()`                 | PlanningPanel      | 地形取得を開始（完了後にHIKINGへ）|
| `on_player_downed(reason)`       | Player (call_group) | ゲームオーバー処理             |
| `on_goal_reached()`              | ゴールArea3D        | 登頂リザルト表示（HIKING中のみ）|

---

## 6. ゴール・天候イベント

- **ゴール**: 山頂に `Area3D`（半径8m円柱）+ 発光する赤い標柱（高さ30m）を動的生成。
  `body_entered` でプレイヤーを検知し `on_goal_reached()` を呼ぶ
- **天候急変**: HIKING 開始から 60〜150 秒（乱数）後に雨。
  `PlayerStats.weather_mult = 1.3` を設定し、HUD にメッセージと天候表示を出す
- **マウス制御**: ESC で解放、HIKING 中のクリックで再キャプチャ
