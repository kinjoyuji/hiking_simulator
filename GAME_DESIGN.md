# ゲームデザインドキュメント（GDD）

ハイキングシミュレーター v0.1 — 初版

---

## 1. ゲーム概要

| 項目 | 内容 |
|------|------|
| タイトル | ハイキングシミュレーター（仮） |
| ジャンル | リアル系登山シミュレーション |
| プラットフォーム | PC / Android |
| エンジン | Godot 4 |
| 開発規模 | 個人〜小規模チーム |

### コアバリュー

1. **リアリズム** — 実際の登山判断を忠実に再現する
2. **学習性** — 遊びながら登山の思考パターンが身につく
3. **緊張感** — 誤った判断がリアルなリスクに直結する

## 2. ゲームループ

```
[計画フェーズ] → [出発フェーズ] → [行動フェーズ] → [帰還 or 緊急フェーズ]
      ↑                                                        |
      └────────────────── リザルト・振り返り ───────────────────┘
```

### 2-1. 計画フェーズ（山行前）

プレイヤーが登山前に行う準備。ここでの判断が全体の難易度に影響する。

- **ルート選定**: 距離・累積標高・技術難易度・コースタイムの確認
- **天気確認**: 出発日・行動中の天気予報（崩れる予報なら中止判断も選択肢）
- **装備チェックリスト**: 必須装備の確認（地図・コンパス・エマージェンシーシート・救急キットなど）
- **タイムプランニング**: 出発時刻・各チェックポイントの通過予定時刻・下山時刻の設定
- **エスケープルートの確認**: 緊急時の撤退ルートを事前に把握

### 2-2. 出発フェーズ

- ザックへのパッキング（容量・重量のバランス）
- 登山届の提出（任意だが、遭難時の救助難易度に影響）

### 2-3. 行動フェーズ（山行中）

ゲームのメイン。プレイヤーはリアルタイムまたはターン制で行動を選択する。

**定期的な判断ポイント:**
- ペース配分（飛ばしすぎると後半に体力不足）
- 水分・食料補給のタイミング
- 天気の変化への対応（引き返す判断）
- 道迷いの検知と対処（現在地の確認方法）
- 時刻と体力を見た引き返し判断（引き返し点の判断）

### 2-4. 緊急フェーズ（トラブル発生時）

特定の条件で発生するイベント。

| トリガー | 内容 |
|----------|------|
| 道外れ | サバイバルフェーズへ移行。現在地特定、ビバーク判断 |
| 転倒・滑落 | 負傷状態。応急処置の選択、行動制限 |
| 天候急変 | 雷・ホワイトアウト・低体温症リスク |
| 体力・水分枯渇 | 行動不能。救助要請の判断 |

## 3. パラメータ設計

### 3-1. プレイヤーステータス

| パラメータ | 範囲 | 説明 |
|-----------|------|------|
| 体力（スタミナ） | 0〜100 | 歩行・荷物重量・傾斜・気温で消耗 |
| 水分 | 0〜100 | 時間・運動量・気温で消耗。不足すると体力消耗増加 |
| 空腹 | 0〜100 | 時間・運動量で消耗。不足すると体力消耗増加 |
| 携帯バッテリー | 0〜100 | 地図アプリ・緊急連絡に使用 |
| 体温 | 35〜38℃ | 気温・装備・行動で変化。低体温症・熱中症リスク |

### 3-2. 状態異常

| 状態 | 発生条件 | 効果 | 回復手段 |
|------|----------|------|----------|
| 捻挫（軽） | 転倒 | 移動速度低下 | テーピング |
| 捻挫（重） | 転倒・悪路 | 移動速度大幅低下 | 固定・安静（行動制限） |
| 裂傷 | 転倒・岩場 | 体力継続消耗 | 止血・包帯 |
| 低体温症（初期） | 体温低下 | 判断力低下（選択肢が減る） | 防寒・行動 |
| 低体温症（重篤） | 体温著低 | 行動不能 | ビバーク・救助要請 |
| 道迷い | ルートを外れる | 現在地不明・精神ストレス上昇 | 地図確認・来た道を戻る |

### 3-3. 装備とパラメータへの影響

**ザック重量による体力消耗倍率:**

| 重量 | 消耗倍率 |
|------|---------|
| 〜5kg | ×1.0 |
| 5〜10kg | ×1.2 |
| 10〜15kg | ×1.5 |
| 15kg〜 | ×2.0 |

**スキルレベル:**
- 歩行技術Lv（1〜5）: 高いほど体力消耗軽減、転倒リスク低下
- 登攀技術Lv（1〜5）: 岩場・鎖場での消耗軽減、滑落リスク低下
- ナビゲーションLv（1〜5）: 高いほど道迷いリスク低下、早期検知

## 4. マップ・ルート設計

### 4-1. マップ生成

- 国土地理院 標高タイルAPI（無料）を使用して実際の地形を3D化
- 対象エリア: 日本国内の主要登山エリア
- LOD（詳細度）: プレイヤー周辺は高解像度、遠景は低解像度

### 4-2. ルート種別

| 難易度 | 想定ルート例 | 必要スキル |
|--------|------------|----------|
| 入門 | 整備された山道（片道2〜3h） | なし |
| 初級 | 累積標高500m程度 | 歩行Lv1 |
| 中級 | 鎖場あり・1泊2日 | 歩行Lv3・登攀Lv1 |
| 上級 | バリエーションルート | 全スキルLv4以上 |

### 4-3. 地形オブジェクト

- 登山道（整備済み・未整備）
- 水場（沢・湧水・山小屋）
- 休憩ポイント（ベンチ・ひらけた場所）
- 危険箇所（ガレ場・鎖場・渡渉）
- 山小屋・避難小屋

## 5. UI / UX 方針

### 5-1. HUD（行動中の表示）

- ステータスバー（体力・水分・空腹・バッテリー）は常時表示
- 時刻・現在標高・気温の表示
- ミニマップ（縮小可能）
- ルートからの逸脱警告

### 5-2. 判断UIのリアリズム

- 地図は実際の地形図形式（等高線・記号）
- コンパスは実際の使い方を模倣
- 天気予報は数値ではなくアイコン・概況で表現（あいまいさを再現）

### 5-3. フィードバック・教育要素

- 行動後に「なぜそれが正解/不正解か」の簡単な解説を表示
- リザルト画面で判断の振り返りを表示
- ヒントモード（ON/OFFで初心者・上級者向けに切替）

## 6. MVP（最初に作るもの）

フルスコープは大きいため、まず以下に絞った動作するプロトタイプを目指す。

**MVP スコープ:**
1. 固定ルート1本（入門コース）
2. 体力・水分の2パラメータのみ
3. 歩行のみ（イベントは天気変化と水分補給のみ）
4. シンプルな3D地形（国土地理院データ or 仮地形）
5. 歩行完了 or 体力/水分枯渇でゲームオーバーのみ

**MVPで検証したいこと:**
- 地形生成パイプラインの技術的実現可能性
- パラメータ消耗バランスの調整感
- 基本的なゲームループの面白さ
